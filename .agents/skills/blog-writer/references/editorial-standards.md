# Editorial Standards

## Reader Promise

Write for students who want a concept to become usable, not merely defined. Every section must advance the reader toward the learning outcome.

## Opening

Start with one of:

- a concrete debugging or interview problem
- a surprising output
- a comparison the reader cannot yet explain
- a direct question with practical consequences

Avoid broad openings such as "In today's fast-paced world," dictionary definitions, and motivational throat-clearing.

## Structure

Use this default sequence when it fits:

1. Problem and reader promise
2. Mental model
3. Small running example
4. Core mechanism
5. Code or worked example
6. Comparison or edge cases
7. Common mistakes
8. Interactive check
9. Practical summary or challenge

Do not force sections that add no value.

## Voice and Pacing

- Prefer direct, conversational technical prose.
- Use concrete nouns and active verbs.
- Vary sentence length without becoming theatrical.
- Keep most paragraphs between one and four sentences.
- Define jargon at first use.
- Use "you" when guiding an action; avoid repeatedly addressing the reader.
- Remove duplicated conclusions and transition filler.
- Use emphasis sparingly.
- Avoid exaggerated claims such as "ultimate," "revolutionary," and "everything you need."

## Examples

- Reuse one dataset, program, or scenario across the article.
- Introduce the smallest example that demonstrates the concept.
- Show expected output where it prevents ambiguity.
- Explain why the example behaves that way.
- Include failure cases when they teach a common misconception.

## Technical Integrity

- Distinguish simplified teaching models from implementation details.
- State assumptions and version-specific behavior.
- Prefer primary sources for current language, framework, database, or API behavior.
- Do not create citations that were not consulted.
- Ensure code is syntactically valid and consistent across snippets.
- Never include credentials, private URLs, or personal data.

## Engagement Test

Before publishing, confirm:

- The first 100 words contain a real problem or question.
- The reader knows what they will learn.
- Every major concept has an example.
- At least one section addresses a likely mistake.
- The quiz tests understanding rather than memorization.
- The conclusion gives a practical next action.
- Removing any paragraph would not make the article stronger.
